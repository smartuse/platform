Welcome!

This is a backend to publish geospatial Data Packages- an application that allows us to connect and manage spatial information, then configure and export it to the frontend applications through an API based on the Frictionless Data standard.

Projects can be accessed and their models and viewed by clicking on the SmartUse logo on the home page. This is for the time beingi the primary navigation between projects. There is also a link on the home page to an online API platform for developers. The frontend is built on this basis, and other apps could also connect to SmartUse with the API. The Web API can be used to access data about Projects.

This project is developed and supported by SmartUse GmbH.

User Guide

The following instructions detail how to manage your data publication in the Smart Use backend.

  1. Start by creating a User for every organization, with links to logos etc. The e-mail address is used to look for a Gravatar image.
  2. Create a Project which defines the study or location of interest. This is where the main documentation about the use case will reside. Markdown formatting is supported for the description.

The following data formats are currently supported:

  • png/gif - a lossless image file
  • jpeg - a lossy compressed image file
  • geojson - Web-optimized geospatial data - see http://geojson.org/
  • datapackage - a geospatial data package - see https://frictionlessdata.io/
  • embed - a public, Web-published embedded map

Data files for any of the formats above can be uploaded in the Data tab, and then linked with the path. For example, if you uploaded a GeoJSON file called pendlerorte.geojson into a test_project/data folder, you could just enter /uploads/test_project/data/pendlerorte.geojson. Just click on the file listing to visit the location at the appropriate URL.
